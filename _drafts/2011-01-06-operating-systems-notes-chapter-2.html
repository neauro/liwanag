Operating systems allow:
  • At low level -- for a process to make requests from the OS directly
  • At high level -- for a user to make a request without writing a program

Low-level requests get their answers from the system-call level, which provides basic functions for controlling processes, and manipulating file and devices. Higher-level requests are satisfied by the command interpreter, and translated really into a sequence of system calls.

System services are classified into categories: program control, status requests, I/O requests. Program errors are also, implicitly, cries for help service.

The state of the computer system and the status of the system's jobs are recorded into various tables.

Operating systems are huge, so modularity is important. Modularity describes the degree to which a system's components can be separated and recombined -- basically, you don't want to be working in one huge chunk all at once, but smaller pieces that you fit together later. You can design an OS in layers, using the virtual-machine concept.
